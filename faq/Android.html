Android Q&A

SDK在Android 6.0系统初始化时崩溃

麻烦请先关闭instant run或者打包应用为apk再安装，再重新测试。如果还有问题，请联系tech-support@vungle.com。

Android onUnableToPlayAd 回调返回的reason都包含什么？

  • "Error launching ad"
  • "Ad already playing"
  • "Vungle Publisher SDK was not successfully initialized - please check the logs"
  • "Only " + getElapsedSeconds() + " of minimum " + getMinDelaySeconds() + " seconds elapsed between ads" //只在设置播放间隔时，才会返回该信息
  • "No cached or streaming ad available"

warning: W/com.vungle.warren.Vungle: Placement is snoozed是什么意思？

Vungle目前还无法做到100%填充率。因此有些时候没有给填充，服务器就会让这台设备休息一会。一般10分钟到半小时后，可以再正常请求广告。

InFeed无填充

Vungle目前还无法做到100%填充率。有些地区填充确实会低一些。测试时可以使用测试App id:5bf49746b1fd5362ddda51e2 和placement reference id:FLEXFEED-2744806 进行测试。切勿上线使用该IDs.

多Auto Cache最佳实践

自从6.4版本后，Vungle已经推出了多AutoCache功能（目前处于Beta测试状态）。但与之相匹配的dashboard设置已经下架，并转为Admin专有功能。如果想开启多AutoCache，请联系Account Manager。 那么开启多Auto Cache后，同时也造成了回调触发点与之前有一些不同。AutoCache的广告位回调会在InitCallback中触发。使用LoadAd方法，LoadAdCallback回调大概率会触发onError的回调。原因是该广告已经自动缓存，处于缓存中。 那么如何解决该问题呢？

最佳方法是引入一个回调，统一缓存成功回调到一个触发点。示例如下：

public interface AdAvailabilityListener {
    void onAdAvailable(String placementId);
}

AdAvailabilityListener mAdAvailabilityListener = new AdAvailabilityListener(){
    @Override
    public void onAdAvailable(final String placementReferenceID) {
        // Unified callback that will fire when any placement has an ad 
        // available to play. Implement any action to be done when ad 
        // becomes available.
    }
};

Vungle.init(app_id, getApplicationContext(), new InitCallback() {
    @Override
    public void onAutoCacheAdAvailable(final String placementReferenceID) {
        mAdAvailabilityListener.onAdAvailable(placementReferenceID);
    }
...
});

private final LoadAdCallback vungleLoadAdCallback = new LoadAdCallback() {
    @Override
    public void onAdLoad(final String placementReferenceID) {
        mAdAvailabilityListener.onAdAvailable(placementReferenceID);
    }
...
};

移除写存储权限

在工程中的manifest中加入以下内容：

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" android:maxSdkVersion="18" tools:node="remove"/>

验证方法：加入改行后，打包APK，用Android Studio分析APK功能，查看APK中的Manifest。
